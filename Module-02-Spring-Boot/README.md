# Module 02 – Spring Boot

## 📖 Module Overview

This module introduced Spring Boot and demonstrated how it simplifies Spring application development through convention over configuration, auto-configuration, starter dependencies, and production-ready features.

The focus was on understanding the purpose of Spring Boot, creating applications quickly, managing configuration, and preparing applications for real-world development.

---

## 📚 Topics Covered

- What is Spring Boot
- Why Spring Boot
- Spring vs Spring Boot
- Spring Boot Architecture
- Creating the First Spring Boot Project
- `@SpringBootApplication`
- `application.properties`
- Spring Boot Configuration
- Spring Profiles (Spring Boot usage)
- Starter Dependencies
- Spring Boot Auto Configuration
- Logging in Spring Boot
- Spring Boot Actuator
- Spring Boot DevTools
- Spring Boot Mini Project

## 💡 Backend Best Practices

- Keep configuration externalized.
- Use meaningful log messages.
- Expose only required Actuator endpoints.
- Use DevTools only during development.
- Avoid unnecessary dependencies.
- Understand defaults before overriding them.
- Read startup logs carefully.
